# Clipper — 超軽量・コンパクトなmacOS用クリップボード履歴マネージャー

Clipperは、macOS向けにSwiftとSwiftUIのみを使用して開発された、極めて軽量・高速・コンパクトなクリップボード履歴マネージャーです。

重いフレームワークやサードパーティ製の巨大なライブラリを一切使用せず、macOSのネイティブAPIのみで構築されているため、圧倒的な低負荷と高いパフォーマンスを実現しています。

---

## 🌟 主な特徴 (Features)

- **圧倒的な軽量・コンパクト設計**
    - **ピュアSwift/SwiftUI製**: アプリ自体のサイズが極めて小さく、起動は一瞬です。
    - **省電力・低負荷**: クリップボード（`NSPasteboard`）の監視は、0.5秒間隔の非同期タスクで行われます。変更が検出された時のみメインスレッド（メインアクター）で処理を行うため、CPUやバッテリーへの負荷を最小限に抑えています。
    - **Dockアイコンの非表示**: 常駐型のユーティリティとして動作するため、Dockにはアイコンを表示せず、メニューバー（ステータスバー）からスマートに管理・操作可能です。
- **Liquid Glassを採用した美しいモダンUI**
    - macOSのシステムと調和する、半透明（グラスモフィズム）の洗練されたパネルUI。
    - 画面端での表示位置の自動調整に対応しています。
- **リッチテキスト対応**
    - プレーンテキストだけでなく、コピー元のRTF（リッチテキスト形式）やHTML形式のスタイルデータも保持するため、フォーマットを崩さずにペーストできます。
- **クイックアクセス＆自動ペースト**
    - **`Command`（⌘）キーのダブルタップ**、またはメニューバーのアイコンクリックでパネルが瞬時にトグル表示されます。
    - 履歴リストから項目を選択するだけで、クリップボードが更新され、**自動で貼り付け（ペースト）**　が実行されます。
- **スマートな履歴管理＆検索**
    - **インクリメンタル検索**: パネル上部の検索バーに入力するだけで、履歴をリアルタイムに絞り込めます。
    - **アプリ情報の取得**: どのアプリケーションからコピーされたかをアイコンや名前で管理。
    - **重複防止**: 直前と同じ内容のコピーは重複して記録されません。
    - **保存件数制限のカスタマイズ**: メニューから保存上限数（50件、100件、500件、1000件、2000件）を変更でき、ストレージの不要な消費を防ぎます。

---

## 💻 動作環境 (System Requirements)

- **OS**: macOS 14.0 (Sonoma) 以上（SwiftUIのLiquid Glass効果 / `.glassEffect` を使用しているため）

---

## 🚀 使い方 (Usage)

1.  **起動**: アプリケーションを起動すると、メニューバーに書面マークのアイコンが表示されます。
2.  **パネルの表示/非表示**: `Command`（⌘）キーを素早く2回押すか、メニューバーのアイコンを左クリックします。
3.  **ペースト**: パネルから履歴を選択すると、現在アクティブなウィンドウへ自動的にペーストされます。
4.  **コンテキストメニュー**: アイコンを右クリックすると、以下のメニューにアクセスできます。
    - 履歴をクリア
    - 保存件数制限の切り替え
    - アプリの終了

## 🛠 コンパイル・ビルド方法 (Compilation & Build)

ClipperはXcodeプロジェクトとして構成されているため、Xcodeを使用する方法とターミナルからコマンドラインを使用する方法の2種類でコンパイルできます。

### 1. Xcodeを使用する場合 (推奨)

1.  **Xcodeの準備**: Mac App StoreなどからXcodeをインストールします。
2.  **プロジェクトを開く**: 本リポジトリのclipper.xcodeprojをXcodeで開きます。
3.  **コンパイルと実行**:
    - メニューの `Product` > `Build`（または `Cmd + B`）でビルドします。
    - `Product` > `Run`（または `Cmd + R`）でビルドしてアプリを即座に実行できます。
4.  **配布用アプリの書き出し (アーカイブ)**:
    - メニューの `Product` > `Archive` を選択します。
    - アーカイブ完了後、オーガナイザウィンドウが表示されるため、`Distribute App` をクリックして、インストール可能な `.app` パッケージとしてローカルに書き出します。

### 2. ターミナル (Command Line) を使用する場合

ターミナルから直接コンパイルして、`.app` アプリケーションファイルを生成することも可能です。

1.  **アクティブなデベロッパディレクトリの設定**
    コマンドラインツールがXcode本体を参照しているか確認します。CommandLineToolsが選択されている場合は、以下のコマンドでXcodeを参照するように切り替えてください。

    ```bash
    sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
    ```

2.  **Releaseビルドの実行**
    リポジトリのルートディレクトリで以下のコマンドを実行し、最適化されたRelease構成でビルドを行います。

    ```bash
    xcodebuild -project clipper.xcodeproj -scheme clipper -configuration Release -derivedDataPath build clean build
    ```

    ビルドが成功すると、`build/Build/Products/Release/clipper.app` にアプリケーションが生成されます。
